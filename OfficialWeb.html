Project: Automation Testing Framework for EFT Switch Application:

DOMAIN : FinTech | Role: QA Automation Engineer | Technologies: Selenium WebDriver, TestNG, Java, Maven, Git, Apache POI
DESCRIPTION : Developed and implemented an automation testing framework for an EFT (Electronic Funds Transfer) Switch application to support regression and smoke testing. The framework was designed with a hybrid approach, integrating data-driven testing capabilities for improved flexibility and reusability.

KEY RESPONSIBILITIES :

  • Designed and set up a custom hybrid automation framework from scratch using Selenium WebDriver, TestNG, and Maven.
  • Implemented group-based test execution for regression suites using TestNG XML configurations.
  • Developed custom TestNG listeners for:
    • Screenshot capture on test failure.
    • Retry logic for flaky test cases using RetryAnalyzer.
    • Enhanced test result reporting with custom XML and POM plugin configurations.
  • Integrated Excel-based DataProvider using TestNG to read test data dynamically. Debugged and fixed issues related to incomplete row reads in Excel files.
  • Utilized Git for version control and collaborated via GitHub on feature branches and merges.
  • Captured and documented test failures with logs and screenshots, aiding in quicker debugging and resolution.
  • Followed Agile practices, participating in regular sprint reviews and stand-ups, while managing tasks in Jira.

System Testing :

The entire environment setup readiness checking which involves -

  • OS(Checking and ensuring the readiness of the Windows and Linux machines on which the SqlServer, front-end application and the back-end applications are running)
  • Checking and Verifying the system network is active and ready for the testing(The ip-address, subnet-mask and firewall settings will be verified to match with that configured in the application softwares)
  • Verifying the Webserver service is running and the UI is available at the endpoint url(UI is hosted in ApacheTomcat)
  • Starting the back-end application(EFT Switch) and verifying that the application server is listening on the configured port.
  • Ensuring that the SQLServer service is runnig and available at the default port(1433).
